What is the purpose of machine learning?¶

To inform decisions based on available data.

Machine learning algorithms are designed to detect patterns in data.

To do this, they have to learn from the data.

Our task:¶

Based on these data, develop a set of rules for determining whether an object is a KN or a SN¶

Any ideas?

This was what I tried...¶

Maybe there exists a set of weights (w0, w1, w2) such that

$$ w_0 \times \textrm{FITPROB_Ia} + w_1 \times \textrm{FITPROB_Ibc} + w_2 \times \textrm{FITPROB_II} $$

is characteristic of whether an object is a KN or an SN.

To make this more explicit, let's include a w3 and say if

$$ w_0 \times \textrm{FITPROB_Ia} + w_1 \times \textrm{FITPROB_Ibc} + w_2 \times \textrm{FITPROB_II} > w_3 $$

then the object is a KN. Otherwise, we'll say our rules classified it as a SN.

To find the set (w0, w1, w2, w3) such that our rule leads to the most accurate prediction of a KN versus a SN, let's just try all the possibilities.

Recap¶

So, we did it! But can we do it in a smarter way?

What are the weaknesses of our approach?

  • We had to specify every single weight combination
  • We were restricted to the weights we specified
  • Golly this was slow when I ran it

A slightly smarter approach to finding the best weights¶

  1. Start with an initial set of weights
  2. For each example in your dataset...
    • apply the current weights to it to get a prediction
    • calcultate the difference between the prediction and the truth
    • update the current weights by a factor corresponding to the difference
  3. Cycle through your dataset as many times as you need to reach convergence

It can be shown mathematically that the difference in step 2b will decrease monotonically with each iteration through the entire dataset.

(It's just one partial derivative and some linear algebra, so it's not scary at all. Ask me offline if you want to get into the math.)

The algorithm we just described can be written in code like this:

def fit(X, y, n_iter, eta): 

    weights = np.zeros(X.shape[1]) 

    # Cycle through your dataset as many times as you want
    for _ in range(n_iter): 

        # Cycle through each example in your dataset
        for xi, target in zip(X, y):

            # Get prediction from current weights
            result_from_applying_weights = np.dot(xi, weights)
            prediction = 1 if result_from_applying_weights > 0 else 0

            # Get the difference between the truth and your prediction
            update = eta * (target - prediction) 

            # Update the current_weights
            weights += update * xi

    return weights

Where X is your feature_matrix from before, y is the CLASS column, n_iter is the number of times you want to cycle through your dataset, and eta controls how much the weights get updated each time.

Example Summary¶

The Perceptron had a few advantages over our "greedy" approach:

  • It didn't need to try all the weights
  • It only spent time trying weights that had a good chance to be the best ones
  • It was able to try weight combinations that we didn't specify ahead of time

Essentially, the Perceptron did the work of finding the best weights on it's own; we just told it two things:

  1. How to decide whether a set of weights was good or bad
  2. How to choose a new set of weights in a favorable way

And that is machine learning.

Based on available data, develop a set of rules to make decisions about new data.

Machine learning best practices¶

With automated rule-finding approaches, there are some weaknesses we have to be aware of:

  1. The rules found by your ML algorithm will always be a function of the training data

  2. The ML algorithm may fit to the noise in your data if you're not careful

So next we'll look at how to utilize a ML algorithm in a way that will generalize well to unseen data.

Why did it suck? Well when the classifier learned the "best classification rules" from the training set, it effectively memorized the training set rather than locking on to the underlying information that will differentiate a KN from a SN.

Training and Validation Sets¶

The ML way to prepare for unseen data is to reserve a portion of your training set for testing. We'll call this portion the "validation set."

Scikit-Learn has a convenient function to do this split for us:

In [24]:
from sklearn.model_selection import train_test_split

X_train, X_validate, y_train, y_validate = train_test_split(X, y, test_size=0.33, stratify=y, random_state=42)

Now let's let the classifier only learn the rules from the training set and see how it performs on the validation set.

In [25]:
dtc.fit(X_train, y_train)

y_validate_pred = dtc.predict(X_validate)

print("Accuracy: ", accuracy_score(y_validate, y_validate_pred))

Accuracy:  0.6593406593406593

You may be disuaded by the low accuracy, but I would argue this has been a success.

We can now use the validation set as a reliable identifier for how well we should expect to perform on new data.

Our next step will be to do something about the low accuracy.

This should be the expected behavior. As you increase the depth of the tree, the tree becomes more likely to memorize the training data. And when the training data is memorized, you will likely perform poorly on new data.

This suggests we can imporve our accuracy by setting max_depth=5.

ML Best Practices Summary¶

You absolutely need to take the steps to make sure your classifier will generalize to new data. You also need to make sure you're doing the best you can possibly do.

You can achieve both by:

  • Reserving parts of your training set for validation
  • Using the validation set to test out different versions of your classifier

Thankfully Sci-Kit learn makes this easy!

Random Forests¶

The RandomForestClassifier is an ensemble of DecisionTreeClassifiers. It's also my personal favorite.

Each decision tree in the ensemble trains on a different subset of the training data, which incorporates cross-validation into your training. This process is called "bagging."

Each sample is also weighted according to how much it contributes to successful classifications or how much it confuses the classifer. This process is called "boosting."

In general, RandomForestClassifiers have the following benefits:

  • Robust against the relative scaling of your data features
  • Resistant to overfitting if you specify a max_depth
  • Built-in multiprocessing
  • Outputs probabilities instead of just predictions

Let's try it out!

k-Nearest Neighbors¶

For a given integer value of k, the classifier will look at each point in your feature space and look at the classes of the nearest k objects in your training set to this point. The classification will be based on a majority vote from those k objects.

Important notes with kNN:

  • You absolutely, postively, 100% must scale your data
  • You will have to develop a way to choose the best value for k

For this dataset, all the feature values are already between 0 and 1, so we're set. Now let's choose the best k.

A Typical ML Workflow¶

Let's choose the RandomForestClassifier as our algorithm and put everything from this talk together.

In [42]:
# Load your training and testing data
train_dataset, test_dataset = pd.read_csv('training_set.csv'), pd.read_csv('testing_set.csv')

# Select the features you want to work with
X_train, y_train = train_dataset[['FITPROB_Ia', 'FITPROB_Ibc', 'FITPROB_II']].values, train_dataset['CLASS'].values
X_test, y_test = test_dataset[['FITPROB_Ia', 'FITPROB_Ibc', 'FITPROB_II']].values, test_dataset['CLASS'].values

# Instantiate a classifer and define the hyperparamters you want the check
rfc = RandomForestClassifier()
param_grid = {'n_estimators': [5, 10, 50, 100],
              'max_depth': [2, 3, 5, 7, 10, None],
              'criterion': ['gini', 'entropy'],
              'min_samples_split': [2, 3, 5, 8, 10],
              'min_samples_leaf': [1, 2, 4, 8],
              'max_leaf_nodes': [5, 10, 20, None]}

# Perform a Cross-Validated GridSearch
gs = GridSearchCV(rfc, param_grid, cv=5, n_jobs=-1)
gs.fit(X_train, y_train)

# Select the best estimator and check how you did on the testing data
print("Accuracy: ", accuracy_score(y_test, gs.best_estimator_.predict(X_test)))

Accuracy:  0.6892177589852009
